# 检查 GitHub Actions 编译状态

## 快速检查清单 ✅

### 1. 检查 Actions 是否启用

- [ ] 仓库 Settings → Actions → General
- [ ] 确保 "Allow all actions" 已启用

### 2. 检查 Workflow 文件

- [ ] 文件路径：`.github/workflows/build.yml`
- [ ] 文件内容格式正确（YAML）

### 3. 检查 Actions 标签页

- [ ] 在仓库页面能看到 "Actions" 标签页
- [ ] 点击后能看到 "Build Termux Binary" workflow
- [ ] 有 workflow 运行记录（即使失败）

### 4. 手动触发测试

- [ ] 点击 "Build Termux Binary"
- [ ] 点击 "Run workflow"
- [ ] 选择分支（main 或 master）
- [ ] 点击 "Run workflow" 按钮
- [ ] 查看是否开始编译

---

## 常见问题

### ❌ 问题：看不到 Actions 标签页

**原因**：Actions 未启用或仓库是新的

**解决**：
1. Settings → Actions → General
2. 启用 Actions
3. 刷新页面

### ❌ 问题：Actions 标签页是空的

**原因**：Workflow 文件不存在或位置不对

**解决**：
1. 检查 `.github/workflows/build.yml` 是否存在
2. 如果不存在，创建它（参考上面的内容）

### ❌ 问题：Workflow 运行但失败

**原因**：编译错误或依赖问题

**解决**：
1. 点击失败的 workflow
2. 查看错误日志
3. 根据错误信息修复

### ❌ 问题：Workflow 没有自动触发

**原因**：触发条件不匹配

**解决**：
1. 确保推送到 main 或 master 分支
2. 或者手动触发（workflow_dispatch）

---

## 测试步骤

1. **上传代码到 GitHub**
2. **检查 Actions 标签页** - 应该自动开始编译
3. **如果没有自动开始** - 手动触发
4. **等待编译完成** - 通常 2-5 分钟
5. **下载文件** - 在 Artifacts 部分
